# MBB Consulting Methodologies Reference

Comprehensive guide to methodologies used by McKinsey, BCG, and Bain (MBB) for professional market research and strategic analysis.

## Core Philosophy: Hypothesis-Driven Approach

All MBB firms use practically identical **hypothesis-driven problem-solving** that places secondary research at the center of analysis. This approach starts with hypothesis formulation instead of aimless data collection, dramatically increasing research efficiency.

**McKinsey principle:** "You don't want to get as much information as possible, you want to get the most important information as quickly as possible."

**Paramount focus:** Quantitative data is paramount – every solution must be backed by solid numbers. Any solution not backed by solid numbers carries heavy burden of proof.

---

## Seven-Step Problem-Solving Process

McKinsey, BCG, and Bain all use this identical 7-step framework that structures secondary research within systematic analysis:

### Step 1: Define the Problem

**Objective:** Precise specification of the key question

**Tools:**
- Problem Statement Worksheet
- SMART criteria (Specific, Measurable, Achievable, Relevant, Time-bound)
- Success criteria definition

**Key questions:**
- What is the precise business question we need to answer?
- What decisions will be based on research results?
- What level of precision and confidence do we need?
- What are time and budget constraints?

**Output:** Clear problem definition that aligns stakeholders before data collection begins

### Step 2: Structure the Problem

**Objective:** Break down complex problems into manageable components

**Tools:**
- MECE principle (Mutually Exclusive, Collectively Exhaustive)
- Logic trees and issue trees
- Hypothesis trees

**Application:** Create structured representation of possible answers to guide subsequent research

**MECE ensures:**
- All problem components are covered (Collectively Exhaustive)
- No duplication of effort (Mutually Exclusive)

### Step 3: Prioritize

**Objective:** Focus effort on high-impact areas

**Tools:**
- Two-by-two matrix: Impact vs. Ease of Implementation
- 80/20 rule application
- Hypothesis prioritization

**McKinsey approach:** Focus on 20% of analysis that delivers 80% of solution value

**Rationale:** Consulting firms charge $200,000/week – efficiency is paramount

### Step 4: Work Plan

**Objective:** Detailed planning before execution

**Define:**
- Specific tasks and analyses required
- Data source identification
- Timeline and milestones
- Responsibility assignments
- Resource requirements

**Research brief:** Comprehensive document aligning stakeholders on desired outcomes

### Step 5: Execute Analyses

**Objective:** Systematic hypothesis testing using secondary sources

**Approach:**
- Quantitative focus – solid numbers for every claim
- Multi-source triangulation
- Continuous validation
- CRAAP test for source quality

**Data hierarchy:** Tier 1 sources (government, peer-reviewed) prioritized over Tier 2-3

### Step 6: Synthesize Findings

**Objective:** Continuous insight generation, not end-loaded analysis

**Tools:**
- Pyramid Principle structuring
- "Day 1 drafts" – immediate synthesis
- Continuous hypothesis refinement

**Critical practice:** Never accumulate synthesis until project end. Create initial insights immediately, refine continuously as new data emerges.

### Step 7: Develop Recommendations

**Objective:** Actionable plan with clear accountability

**Components:**
- Specific action items
- Clear ownership assignments
- Measurable success criteria
- Implementation timeline
- Risk mitigation strategies

**Standard:** Recommendations must be specific, actionable, and prioritized

## MECE Principle (Mutually Exclusive, Collectively Exhaustive)

Fundamental pillar of McKinsey thinking developed in the 1960s-70s.

### Mutually Exclusive

**Definition:** Components do not overlap
**Purpose:** Eliminates duplicate effort and redundant analysis
**Application:** Each data source, analysis component, or insight category should be distinct

**Example:** When analyzing market segments, ensure no customer falls into multiple segments

### Collectively Exhaustive

**Definition:** All relevant components are covered
**Purpose:** Ensures nothing important is missed
**Application:** Systematic coverage of all relevant data sources without gaps

**Example:** When researching competitors, ensure all significant market players are included

### MECE in Secondary Research

**Source organization:**
- Tier 1, 2, 3 categorization ensures comprehensive coverage (Exhaustive)
- Each source provides unique information without duplication (Exclusive)

**Analysis structure:**
- Four pyramid levels (Segment, Customers, Competitors, Company) are mutually exclusive
- Together they cover all strategic positioning aspects (collectively exhaustive)

## Pyramid Principle

Developed by Barbara Minto at McKinsey in the 1960s. Structures thinking from details to main conclusions.

### Core Concept

**Bottom-up thinking:** Start with detailed facts, build to supporting insights, culminate in main conclusions
**Top-down communication:** Present main conclusion first, then supporting arguments, then detailed evidence

### Structure

```
                    [Main Conclusion]
                           |
        +------------------+------------------+
        |                  |                  |
   [Support 1]        [Support 2]        [Support 3]
        |                  |                  |
    +---+---+          +---+---+          +---+---+
    |   |   |          |   |   |          |   |   |
  [Facts] [Facts]    [Facts] [Facts]    [Facts] [Facts]
```

### Application to Market Research

**Data collection phase (bottom-up):**
1. Collect individual data points (facts)
2. Group related facts into insights
3. Synthesize insights into conclusions
4. Build main recommendation from conclusions

**Presentation phase (top-down):**
1. Lead with main recommendation
2. Support with 3-5 key conclusions
3. Provide insight-level details
4. Reference fact-level data in appendix

### MECE Integration

Each level of pyramid should be MECE:
- Main conclusion has 3-5 mutually exclusive supporting arguments
- Each supporting argument has collectively exhaustive evidence
- No overlap between branches, complete coverage within branches

## Issue Trees and Logic Trees

### Issue Trees

**Purpose:** Disaggregate complex problem into specific questions

**Structure:**
```
Main Question
├── Sub-question 1
│   ├── Component A
│   └── Component B
├── Sub-question 2
│   ├── Component C
│   └── Component D
└── Sub-question 3
    ├── Component E
    └── Component F
```

**MECE application:** Each level must be mutually exclusive and collectively exhaustive

**Example for market entry:**
```
Should we enter this market?
├── Is the market attractive?
│   ├── What is market size and growth?
│   └── What is competitive intensity?
├── Can we compete effectively?
│   ├── Do we have required capabilities?
│   └── Can we achieve cost advantage?
└── Is financial return adequate?
    ├── What is investment required?
    └── What is expected ROI timeline?
```

### Hypothesis Trees

**Purpose:** Structure possible answers to guide research

**Difference from issue trees:** Hypothesis trees propose potential answers, issue trees ask questions

**Example:**
```
Hypothesis: Market is attractive for entry
├── Market size >$1B and growing >10% CAGR
├── Fragmented competition with no dominant player
└── Strong customer need for innovation
```

**Research execution:** Test each hypothesis branch systematically with secondary data

## Analytical Frameworks

### BCG Growth-Share Matrix (1970)

**Purpose:** Portfolio analysis and resource allocation

**Dimensions:**
- X-axis: Relative Market Share (High to Low)
- Y-axis: Market Growth Rate (High to Low)

**Quadrants:**
1. **Stars** (High growth, High share) – Invest for growth
2. **Cash Cows** (Low growth, High share) – Harvest for cash
3. **Question Marks** (High growth, Low share) – Selective investment
4. **Dogs** (Low growth, Low share) – Divest or harvest

**Secondary research application:**
- Market growth rates from industry reports (Tier 1-2 sources)
- Market share data from company financials and analyst reports
- Cross-industry perspective using publicly available data

### Porter's Five Forces

**Purpose:** Industry attractiveness assessment

**Five forces:**
1. **Threat of new entrants** – Barriers to entry analysis
2. **Bargaining power of suppliers** – Supplier concentration and switching costs
3. **Bargaining power of buyers** – Customer concentration and alternatives
4. **Threat of substitutes** – Alternative solution availability
5. **Competitive rivalry** – Industry concentration and competitive intensity

**Secondary research sources:**
- Industry structure: Trade associations, analyst reports
- Supplier/buyer dynamics: Value chain analysis, company filings
- Substitute threats: Patent databases, technology trend reports
- Competitive intensity: Market share data, pricing trends

### McKinsey 7S Framework

**Purpose:** Organizational effectiveness analysis

**Seven elements:**
- **Strategy** – Plan for achieving competitive advantage
- **Structure** – Organizational hierarchy and reporting
- **Systems** – Processes and procedures
- **Shared Values** – Core beliefs and culture
- **Style** – Leadership approach
- **Staff** – People and capabilities
- **Skills** – Distinctive competencies

**Application:** Assess internal company positioning (Pyramid Level 1)

## 80/20 Rule (Pareto Principle)

### Core Concept

**80/20 observation:** 80% of effects come from 20% of causes

**McKinsey application:** Focus on 20% of analysis that provides 80% of solution value

### Implementation

**Research prioritization:**
- Identify highest-impact questions (Step 3: Prioritize)
- Focus data collection on critical hypotheses
- Accept "good enough" for low-impact questions

**Efficiency standard:**
- Top consulting projects: $200,000/week fees
- Client expects efficient, high-value analysis
- No time for comprehensive coverage of low-impact areas

**Practical application:**
- 3-5 competitors analyzed deeply vs. comprehensive market scan
- Key trends quantified vs. exhaustive trend inventory
- Critical sources validated rigorously vs. every source CRAAP tested

## Continuous Synthesis: "Day 1 Drafts"

### Core Practice

**Traditional approach (WRONG):** Collect all data → Analyze at end → Write report
**MBB approach (RIGHT):** Synthesize continuously from Day 1

### "Day 1 Draft" Concept

**Immediate synthesis:**
- Create initial insights document on first day of research
- Update continuously as new data emerges
- Refine hypotheses in real-time
- Identify gaps and redirect research quickly

**Benefits:**
- Early identification of dead-end paths
- Faster course correction
- Continuous stakeholder alignment
- Reduced end-of-project pressure

### Implementation

**Daily practices:**
1. End each day with synthesis session (30 minutes)
2. Update pyramid analysis document with new findings
3. Refine hypothesis tree based on evidence
4. Identify next-day research priorities
5. Flag high-confidence vs. low-confidence conclusions

**Weekly practices:**
1. Stakeholder check-in with current synthesis
2. Hypothesis validation checkpoint
3. Scope refinement if needed

## Best Practices Summary

### Hypothesis-First Approach

✓ Form hypothesis early, test systematically
✓ Eliminate dead-end paths quickly
✓ Never start data collection without clear hypothesis

### MECE Application

✓ Ensure complete coverage without duplication
✓ Every source should add unique value
✓ Structure all analysis using MECE logic

### Multi-Source Triangulation

✓ Validate critical findings with 2-3 independent sources
✓ If findings converge, confidence is high
✓ If findings diverge, investigate discrepancies

### Continuous Synthesis

✓ Create "Day 1 drafts" immediately
✓ Update insights continuously
✓ Never accumulate synthesis until project end

### 80/20 Efficiency

✓ Focus on 20% of analysis delivering 80% of value
✓ Accept "good enough" for low-impact areas
✓ Prioritize ruthlessly using impact vs. effort matrix

### Epistemic Humility

✓ Explicitly state what is uncertain or unknown
✓ Confidence levels for all conclusions
✓ Methodology limitations acknowledged

## Common Consulting Phrases and Their Meanings

- **"Fact-based"** – Every claim backed by data and citations
- **"So what?"** – What does this insight mean for the client decision?
- **"Structure first"** – Apply MECE before deep analysis
- **"Hypothesis-driven"** – Start with proposed answer, test systematically
- **"Day 1 answer"** – Initial synthesis, continuously refined
- **"80/20"** – Focus on high-impact, deprioritize low-impact
- **"Triangulate"** – Validate with multiple independent sources
- **"MECE it"** – Restructure for mutual exclusivity and collective exhaustiveness
